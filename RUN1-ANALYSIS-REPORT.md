## Native Framework Strategy

**Recommended: Capacitor 6**

Reasons:
- Single `install.html` maps directly to `webDir: 'fleet-apex-flat'`
- No React/Vue build step needed — deploy vanilla JS as-is
- Two separate Capacitor projects: `fleet-apex-admin` + `fleet-apex-driver`
- Same source `install.html` with `?app=admin` and `?app=driver` start URLs
- Capacitor plugins replace all browser APIs that don't work in WebView

**Desktop: Tauri 2**
- Separate Tauri project pointing to same `install.html`
- Lightweight, Rust-based, much smaller than Electron
- Supports Windows (.exe + MSI), macOS (.app + .dmg), Linux (.AppImage)

## Migration Risks

| Severity | Risk | Mitigation |
|----------|------|-----------|
| HIGH | ServiceWorker unsupported in Capacitor WebView | Replace with Capacitor Filesystem caching |
| HIGH | BroadcastChannel limited in single WebView | Use Capacitor app events + Supabase realtime |
| MEDIUM | localStorage 5MB limit | Capacitor Preferences + SQLite |
| MEDIUM | Camera not wired | Add Capacitor Camera to hazard report flow |
| LOW | MapLibre CDN | Bundle maplibre-gl.js in flat directory |
| LOW | No CSP headers | Add meta CSP to install.html |
| LOW | No PIN/biometric vault | Add Capacitor SecureStorage |
